## Kinethestic Learning Activities Collection
We have collected here a list of KLAs for your perusal and use. We only ask that if you have developed any of your own KLAs that you describe them here for everyone's benefit.  To add your own KLA, consider following the KLA [Design pattern](Design_pattern.md).

(Also, see the activities constructed at the [SIGCSE 2004](SIGCSE_2004.md) special session.)

## List Of Activities

1. **[Cons Cell Jeopardy](Cons_Cell_Jeopardy.md)**: Combination of Jeopardy, Twister and Scheme Cons Cells (Andrew Begel)

2. **[Counting the Class](Counting_the_Class.md)**: Randomized algorithm class counting exercise (Steven Wolfman)

3. **[Critical Smurf](Critical_Smurf.md)**: Chaotically dancing Smurfs stabilize to dancing one at a time (Gerard Tel) 

2. **[Data Processing](Data_Processing.md)**: Lining up according to feelings on a continuum

2. **[Disk Cache Latency Relay](Disk_Cache_Latency_Relay.md)**: Motivates why caches are both good and bad (Kate Deibel)

2. **[Distributed Human Monty Hall Experiment](Distributed_Human_Monty_Hall_Experiment.md)**: Students perform the Monty Hall problem, record their results, and then aggregate the results to investigate the base problem

2. **[Do Nothing Why Pipeline](Do_Nothing_Why_Pipeline.md)**: Students self-discover the need for pipelining (Kate Deibel)

2. **[Flow Chart Hopscotch](Flow_Chart_Hopscotch.md)**: Students move through a giant flowchart to internalize process control (Rebecca Bates)

2. **[Human Binary Tree](Human_Binary_Tree.md)**: Forming a binary tree out of an entire class and performing calculations with it

2. **[Human Mystery Interface](Human_Mystery_Interface.md)**: Explore interface design issues by acting out a broken interface

2. **[Massive Boolean Short Circuit](Massive_Boolean_Short_Circuit.md)**: Short-circuited Boolean operators

2. **[Mob Topological Sort](Mob_Topological_Sort.md)**: Introduces graph concepts like topological sort and cycle-detection

2. **[Mystery Machine Schedule](Mystery_Machine_Schedule.md)**: Participants make requests for using an imaginary machine, and the teacher uses this input to explain a Greedy Choice Property (Gerard Tel)

2. **[Network Routing on Strings](Network_Routing_on_Strings.md)**: Routing paper packets across string network links (Tammy [VanDeGrift](VanDeGrift.md) and Sarah Schwarm)

2. **[Object Acrobats](Object_Acrobats.md)**: Acrobat and actor exercise for teaching inheritance (Tammy [VanDeGrift](VanDeGrift.md))

2. **[Parameter Passing](Parameter_Passing.md)**: Frisbees and functions and value and reference

2. **[Parameterized Student Sort](Parameterized_Student_Sort.md)**: sort students at the front of the class using a parameterized comparator; good for interfaces in OO languages or higher-order functions in functional languages (Steve Wolfman)

2. **[Sense of Scale](Sense_of_Scale.md)**: Give people a sense of the size of historical computers by showing how much of the class they'd cover.

2. **[Social Sort](Social_Sort.md)**: Non-determinism explained by bilateral card sorting (Gerard Tel)

2. **[Sorting Cards](Sorting_Cards.md)**: Sorting activity with cards (Tammy [VanDeGrift](VanDeGrift.md))

2. **[Sorting Students](Sorting_Students.md)**: The classic front-of-the-room sorting exercise
 
2. **[Spotty Dotty the Continuation Dalmatian](Spotty_Dotty_the_Continuation_Dalmatian.md)**: continuations explained via flying Beanie Babies; also a good classroom atmosphere helper 

2. **[Start of Class Shout](Start_of_Class_Shout.md)**: First day exercise to establish an open classroom atmosphere
 
2. **[Tail Recursion](Tail_Recursion.md)**: Frisbee toss for functions that are recursive vs. tail recursive

2. **[The Function Game](The_Function_Game.md)**: Active parameter passing and interaction with memory (Rebecca Bates)
